# Progest 要件定義書

**バージョン:** 1.0
**作成日:** 2026-02-15
**最終更新:** 2026-02-15

## 1. プロジェクト概要

### 1.1 プロジェクト名
Progest - クリエイター向け資産管理ツール

### 1.2 プロジェクトの目的
クリエイターがプロジェクト内のファイルや資産を一貫した命名規則で管理し、効率的に整理・検索・運用するためのツールを提供する。

### 1.3 ターゲットユーザー
- コンテンツクリエイター（動画、音声、デザイン等）
- ソフトウェア開発者
- プロジェクトマネージャー
- チームでファイルを共有するユーザー

### 1.4 提供インターフェース
- **CLI**: コマンドラインインターフェース
- **GUI**: デスクトップアプリケーション（Avalonia UI）
- **TUI**: ターミナルユーザーインターフェース

## 2. 機能要件

### 2.1 命名規則システム（MVP）

#### 2.1.1 基本的な命名規則変換
ユーザーは以下の命名規則をファイル名に適用できる：

| 規則 | 例 | 優先度 |
|------|-----|--------|
| snake_case | my_file_name | 必須 |
| camelCase | myFileName | 必須 |
| PascalCase | MyFileName | 必須 |
| kebab-case | my-file-name | 必須 |
| Title Case | My File Name | 必須 |

#### 2.1.2 プレフィックス（Prefix）
ユーザーはファイル名の先頭にプレフィックスを追加できる：

**日付ベースのプレフィックス:**
- ISO 日付: `20250215_filename`
- ISO 日付時刻: `20250215_1430_filename`
- リバース日付: `2025-02-15_filename`
- 短い日付: `250215_filename`

**固定文字列プレフィックス:**
- ユーザー定義の固定文字列（例: `img_`, `doc_`, `video_`）

#### 2.1.3 サフィックス（Suffix）
ユーザーはファイル名の末尾にサフィックスを追加できる：

**バージョンベースのサフィックス:**
- セマンティックバージョニング: `filename_v1.0.0`
- シンプルバージョニング: `filename_v2.5`
- シーケンシャル番号: `filename_042`

**固定文字列サフィックス:**
- ユーザー定義の固定文字列（例: `_final`, `_draft`, `-edited`）

#### 2.1.4 シーケンシャル番号付け
バッチ処理で複数のファイルを一括リネームする際、連続した番号を付与：

- 開始番号の指定（例: 0, 1, 100）
- 桁数の指定（例: 3桁: 001, 4桁: 0001）
- セパレータの指定（例: `_`, `-`, `.`）
- 適用例: `0001_file`, `0002_file`, `0003_file`

#### 2.1.5 一括リネーム機能
複数のファイルに対して命名規則を一括適用：
- ディレクトリ内の全ファイルまたは選択したファイル
- シーケンシャル番号の自動付与
- プレビュー機能で変更内容を確認

### 2.2 アセット管理機能

#### 2.2.1 メタデータ管理
各ファイルに以下のメタデータを付与：
- タグ（Tag）
- 説明（Description）
- 作成者（Creator）
- 作成日（Created Date）
- 更新日（Modified Date）

#### 2.2.2 タグシステム
- タグの追加、削除、編集
- タグによる検索
- タグの階層化（親子関係）
- タグの色分け

#### 2.2.3 依存関係管理
- ファイル間の依存関係の追跡
- 依存関係の可視化
- 依存するファイルを含む削除防止
- 依存関係の警告表示

#### 2.2.4 サイドカーファイル
Unityライクなサイドカーファイル（`.meta`ファイル）によるメタデータ管理：
- JSON形式のサイドカーファイル
- ファイルと同名の`.meta`ファイルとして保存
- メタデータの外部保存（ファイル自体に変更を加えない）

#### 2.2.5 データベース連携
- SQLiteによる高速インデックス作成
- サイドカーファイル + データベースのハイブリッド方式
- 全文検索対応
- 高速なフィルタリングとソート

### 2.3 プロジェクト管理機能

#### 2.3.1 プロジェクト定義
- プロジェクト単位で命名規則を管理
- プロジェクト設定の保存と読み込み
- 複数プロジェクトの切り替え

#### 2.3.2 ディレクトリルール
- ディレクトリ単位での命名規則適用
- ネストされたディレクトリ構造のサポート
- ディレクトリレベルのメタデータ

#### 2.3.3 テンプレートシステム
- プロジェクト構造と設定をテンプレートとしてエクスポート
- テンプレートのインポートと適用
- チーム内でのテンプレート共有

### 2.4 AI機能（将来的な実装）

#### 2.4.1 自動命名規則生成
- 既存のファイルパターンから命名規則を学習
- 新しいファイルに適切な命名規則を自動提案

#### 2.4.2 自動タグ付け
- ファイル内容に基づく自動タグ提案
- パターン認識によるタグ付け

#### 2.4.3 スマートファイル整理
- ファイルの内容や種類に基づく自動分類
- 最適なディレクトリ構造の提案

## 3. 非機能要件

### 3.1 パフォーマンス
- 10,000ファイルのプロジェクトで1秒以内の検索応答
- 一括リネームで1分間に1,000ファイル以上処理
- 起動時間は2秒以内

### 3.2 ユーザビリティ
- 直感的なUI/UX
- キーボードショートカットの充実
- 拡張可能なキーバインド設定
- ダークモード対応

### 3.3 信頼性
- データのバックアップ機能
- 操作の取り消し（Undo）/やり直し（Redo）
- 例外処理とエラーメッセージの表示
- データ破損時のリカバリ機能

### 3.4 保守性
- モジュール化されたアーキテクチャ
- テストカバレッジ80%以上
- コードドキュメントの整備
- 変更ログの作成

### 3.5 拡張性
- プラグインシステムによる機能拡張
- カスタム命名規則の定義
- 外部ツールとの連携

## 5. データモデル

### 5.1 命名規則（NamingConvention）
```
- Id: GUID
- Name: string
- Type: ConventionType (enum)
- PrefixRule: PrefixRule (value object)
- SuffixRule: SuffixRule (value object)
- SequentialNumberingRule: SequentialNumberingRule (value object)
```

### 5.2 アセット（Asset）
```
- Id: GUID
- FilePath: string
- FileName: string
- Tags: List<Tag>
- Metadata: AssetMetadata (value object)
- Dependencies: List<FileDependency>
- CreatedAt: DateTime
- UpdatedAt: DateTime
```

### 5.3 タグ（Tag）
```
- Id: GUID
- Name: string
- Color: string (hex)
- ParentTagId: GUID? (nullable)
```

### 5.4 プロジェクト（Project）
```
- Id: GUID
- Name: string
- RootPath: string
- Settings: ProjectSettings
- DirectoryRules: List<DirectoryRule>
- CreatedAt: DateTime
```

## 6. ユースケース

### 6.1 命名規則の適用
1. ユーザーは命名規則を作成する
2. ファイルまたはディレクトリを選択する
3. 適用ボタンをクリックする
4. システムはプレビューを表示する
5. ユーザーが確認すると、ファイル名が変更される

### 6.2 アセットの検索
1. ユーザーは検索バーにキーワードを入力する
2. またはタグでフィルタリングする
3. システムは一致するファイルを高速に表示する
4. ユーザーはファイルをダブルクリックして開く

### 6.3 プロジェクトの初期化
1. ユーザーはCLIまたはGUIで`progest project init`を実行する
2. プロジェクト名とルートパスを指定する
3. デフォルトの命名規則を選択する
4. システムはプロジェクト構造を作成する

## 10. 用語集

| 用語 | 説明 |
|------|------|
| 命名規則（Naming Convention） | ファイル名の命名パターンやルール |
| サイドカーファイル（Sidecar File） | ファイルに関連付けられたメタデータを格納する別ファイル（`.meta`ファイル） |
| セマンティックバージョニング | `major.minor.patch`形式のバージョン管理方式 |
| シーケンシャル番号付け | 連続した番号をファイル名に付与する方式 |
| 集約ルート（Aggregate Root） | DDDにおけるエンティティのグループを管理するルートオブジェクト |
| 値オブジェクト（Value Object） | 属性の値によって同一性が決まるオブジェクト |
