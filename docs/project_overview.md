# プロジェクト概要

`Progest`は、クリエイターのためのアセット管理ツールです。特にプロジェクト間で命名規則を統一するシステムを主軸にしているツールです。

## 提供形式
- CLI
- GUI
- TUI

## 技術スタック（簡易的）
- 言語: C# (必要であればRust)
- フレームワーク: .NET Core
- UIフレームワーク: Avalonia UI
- テーマ: ShadUI
- データベース: SQLite

## 機能
- プロジェクト、ディレクトリ間で命名規則を揃えられる
  - 複雑な命名規則も作れる
    - 基本的な命名規則を決めれる: `snake_case` `camelCase` `PascalCase` `kebab-case` `Title Case` など
    - suffixやprefixを付けられる: prefixとして日付をつけたり　suffixとしてバージョン情報を付けたり
    - 連番機能も提供: `0001_...` `0002_...` とつけれる
- ディレクトリ構造、設定をテンプレートとして書き出し、再利用ができる
  - 新規作成時に利用したり、チームで共有したりが可能
- Unityライクなメタデータを用いたアセット管理(サイドカー方式)
  - タグのサポート
    - タグを追加、削除、編集できる
    - タグを検索、フィルタリングできる
  - ディレクトリにもメタデータを持たせる 
    - 事前によく入れるファイルの拡張子を設定しておくことでファイルのインポート時に提案できるように
    - ディレクトリにも作成した命名規則をデフォルト（プロジェクトの）命名規則と別に設定できる
  - ファイル感の依存関係の整理が可能
    - ファイル間の依存関係を表示できる
    - ファイルの削除時に警告が出てくる
- 前述のサイドカー方式のメタデータに加え、データベース駆動による検索などの高速化（ハイブリッド）
  - `sqlite`によるインデックスと、高速検索、アセット管理。
- AIによる自動命名規則の生成や、ファイルの自動整理機能。テンプレート生成
  - 現在の命名規則に合わせて、生成、提案する。
