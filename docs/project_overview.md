# プロジェクト概要

`Progest`は、クリエイターのためのアセット管理ツールです。特にプロジェクト間で命名規則を統一するシステムを主軸にしているツールです。

## 提供形式
- **CLI** (Command Line Interface) - コマンドラインインターフェース
- **GUI** (Graphical User Interface) - デスクトップアプリケーション
- **TUI** (Terminal User Interface) - ターミナルユーザーインターフェース

## 技術スタック
- **言語**: C# (.NET 10.0)
- **UIフレームワーク**: Avalonia UI
- **MVVMライブラリ**: CommunityToolkit.Mvvm
- **データベース**: SQLite + Entity Framework Core 9.0
- **テスト**: xUnit, FluentAssertions, Moq
- **テーマ**: ShadUI（将来的な実装）

## 開発アプローチ
- **アーキテクチャ**: クリーンアーキテクチャ + DDD（ドメイン駆動設計）
- **開発方法**: TDD（テスト駆動開発）
- **コミット戦略**: 機能単位の細かいコミット
- **進行方法**: バックエンド完遂型（Core → Db → Cli/Tui/Gui）

## 開発方針

### TDD（テスト駆動開発）
1. まずテストを書く
2. テストを失敗させる（Red）
3. テストを通す最小限の実装を行う（Green）
4. コードをリファクタリングする（Refactor）

### 機能単位の細かいコミット
- 1つのコミットで1つの機能に集中
- コミットメッセージは明確で記述的に
- テストと実装を同じコミットに含める

### 不明点は必ず確認
- 要件が不明確な場合は必ずユーザーに確認
- 勝手に機能を追加しない
- 実装前に設計を確認
